# faith-app-content

Версионируемый каталог святоотеческой литературы для Faith App. Тексты в
общественном достоянии; источник текстов указан внутри файлов книг.

`manifest.json` содержит метаданные и SHA-256 каждого файла. Книги публикуются
как assets GitHub Release; приложение проверяет контрольную сумму перед чтением
и сохранением файла.

## Публикация обновления

1. Обновить JSON книги и увеличить поле `version`.
2. Обновить `size_bytes`, `download_url`, `sha256` и `last_updated` в манифесте.
3. Проверить JSON, число глав и соответствие контрольной суммы.
4. Создать новый тег и приложить к релизу `manifest.json` и изменённые книги.

Подготовка релиза `v1.0.2` из проверенных исходных JSON:

```sh
python3 tools/prepare_release.py --source-dir /path/to/content_staging
python3 tools/prepare_release.py --validate-only
```

Скрипт проверяет исходные SHA-256, отделяет сноски от основного
текста, удаляет служебные хвосты и сверяет манифест с готовыми файлами.

## Импорт новых книг и тесты

Три книги релиза `v1.1.0` собираются из дореволюционных изданий,
указанных в метаданных каждого JSON:

```sh
python3 tools/import_patristic_books.py --book all --update-manifest
python3 -m unittest tools.test_import_patristic_books
python3 tools/audit_book_tests.py
python3 tools/prepare_release.py --validate-only
```

Авторские тесты хранятся отдельно в `content_tests/`, а импортёр встраивает их
в `chapters[].test`. Поэтому приложение получает текст и тесты одним файлом и
проверяет их общей контрольной суммой. Для каждого вопроса обязательны три
варианта, ровно один `correct: true` и содержательное `explanation`.

Если электронная публикация содержит спорную атрибуцию, оговорка сохраняется
в `editorial_note` книги и `attribution_note` конкретной главы. Такие сведения
нельзя удалять ради более простого отображения каталога.

Старые релизы не перезаписываются: неизменённые книги могут оставаться по URL
предыдущей версии, а обновлённые получают URL нового релиза.
