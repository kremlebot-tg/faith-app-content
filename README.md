# faith-app-content

Версионируемый каталог святоотеческой литературы для Faith App. Тексты в
общественном достоянии; источник текстов указан внутри файлов книг.

`manifest.json` содержит метаданные и SHA-256 каждого файла. Книги публикуются
как assets GitHub Release; приложение проверяет контрольную сумму перед чтением
и сохранением файла.

## Публикация обновления

1. Обновить JSON книги; при изменении самого текста увеличить поле `version`.
2. Обновить `size_bytes`, `download_url`, `sha256` и `last_updated` в манифесте.
3. Проверить JSON, число глав и соответствие контрольной суммы.
4. Создать новый тег и приложить к релизу `manifest.json` и изменённые книги.

Подготовка релиза `v1.0.2` из проверенных исходных JSON:

```sh
python3 tools/prepare_release.py --source-dir /path/to/content_staging
python3 tools/prepare_release.py --validate-only
```

Скрипт проверяет исходные SHA-256, отделяет сноски от основного
текста, удаляет служебные хвосты и сверяет манифест с готовыми файлами.

## Импорт новых книг и тесты

Три книги релиза `v1.1.0` собираются из дореволюционных изданий,
указанных в метаданных каждого JSON:

```sh
python3 tools/import_patristic_books.py --book all --update-manifest
python3 -m unittest discover -s tools -p 'test_*.py'
python3 tools/audit_book_tests.py
python3 tools/prepare_release.py --validate-only
```

Авторские тесты хранятся отдельно в `content_tests/`, а импортёр встраивает их
в `chapters[].test`. Поэтому приложение получает текст и тесты одним файлом и
проверяет их общей контрольной суммой. Для каждого вопроса обязательны три
варианта, ровно один `correct: true` и содержательное `explanation`.

Большие книги готовятся редакционными партиями в `content_tests/drafts/`.
Частичный черновик не встраивается в книгу и не попадает в релиз, но проходит
те же проверки вопросов, длины вариантов и распределения верных ответов:

```sh
python3 tools/audit_draft_tests.py \
  content_tests/drafts/feofan_dukhovnaja_zhizn_01_10.json
```

Для полного комплекта тестов к «Точному изложению православной веры»
воспроизводимо собирается пакет человеческой богословской рецензии:

```sh
python3 tools/build_damaskin_review_packet.py
```

Пакет сохраняется в `reviews/ioann_damaskin/`, фиксирует commit и контрольные
суммы входных файлов и делит 300 вопросов на четыре проверяемых блока.

Таким же способом собирается пакет к «Приношению современному монашеству»:

```sh
python3 tools/build_ignatius_review_packet.py
```

Он сохраняется в `reviews/ignatij_prinoshenie/`, не перезаписывает редакторские
предостережения в `00_README.md` и делит 180 вопросов на четыре блока по 15
глав.

Для постпубликационной проверки 240 тестов к «Духовной жизни» святителя
Феофана собирается отдельный пакет:

```sh
python3 tools/build_feofan_spiritual_review_packet.py
```

Перед сборкой генератор требует полного совпадения тестов в книге,
объединённом файле и 16 редакционных черновиках.

Оставшиеся три опубликованные книги объединены в ещё один пакет:

```sh
python3 tools/build_early_fathers_review_packet.py
```

Он охватывает 204 вопроса к творениям святителя Афанасия Великого,
преподобного Макария Великого и циклу бесед о покаянии, сохраняя
атрибуционные оговорки исходных JSON.

Полнота всех пакетов контролируется единым реестром:

```sh
python3 tools/audit_review_coverage.py
```

Актуальный снимок находится в `reviews/00_REGISTRY.md`. Наличие пакета не
считается завершённой человеческой рецензией: её вердикт должен быть отдельно
зафиксирован в репозитории.

Если основной текст книги уже опубликован и проверен, тесты можно встроить без
повторной загрузки источника:

```sh
python3 tools/embed_book_tests.py \
  --book ioann_lestvichnik \
  --release-tag v1.3.0 \
  --release-date 2026-07-22 \
  --update-manifest
```

Скрипт сохраняет метаданные, внутреннюю версию, заголовки, абзацы, ссылки и
сноски существующего JSON; изменяются только `chapters[].test` и релизные поля
манифеста. Это позволяет воспроизводимо выпускать обновления тестов, даже если
исходный сайт временно недоступен.

Каждый содержательный раздел книги должен иметь ровно три вопроса. Структурный
заголовок без собственного текста, редакторское послесловие или отдельный
богослужебный текст можно не тестировать только через явный список
`excluded_chapters` с непустой причиной для каждого номера. Аудитор требует,
чтобы все главы были либо проверены, либо документированно исключены, и не
позволяет встроить тест в исключённый раздел.

Если электронная публикация содержит спорную атрибуцию, оговорка сохраняется
в `editorial_note` книги и `attribution_note` конкретной главы. Такие сведения
нельзя удалять ради более простого отображения каталога.

Старые релизы не перезаписываются: неизменённые книги могут оставаться по URL
предыдущей версии, а обновлённые получают URL нового релиза.

Известные слитые цифровые маркеры сносок в опубликованных книгах очищаются
отдельными миграциями с проверкой SHA-256 и точной последовательности маркеров:

```sh
python3 tools/clean_feofan_put.py
python3 tools/clean_feofan_dukhovnaja.py
python3 tools/clean_lestvitsa.py
python3 tools/clean_ignatij_prinoshenie.py
python3 tools/redistribute_ignatij_notes.py
python3 tools/mark_feofan_section.py
```

Это одноразовые миграции для зафиксированных исходных версий. На уже очищенном
файле они намеренно завершаются ошибкой контрольной суммы, не переписывая его.
Сами примечания при этом остаются в `chapters[].notes`, а встроенные тесты и
ссылки не изменяются.

Структурный заголовок без собственного текста хранится с явным
`kind: "section"`, а его номер дублируется в `manifest.json` как
`section_numbers`. Такой элемент показывается в оглавлении, но не открывается
как пустая глава и не входит в прогресс чтения.

Для «Приношения современному монашеству» после очистки нужна ещё одна
одноразовая миграция. Она раскладывает 781 сохранённое примечание по 55
главам на основе зафиксированной карты якорей из коммита `2bfb8a9`, не меняя
заголовки, абзацы, ссылки и другие поля.
